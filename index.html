Project 3: SSP Project – Refinement and Ethical Integration

Author: Tsogt Enkhbat
Date: July 2025

GitHub Repository: https://github.com/heytoshi/SSP-Project

Project Overview:
The SSP Project is a simple social media web app that I built two years ago using React.js and Node.js. It is similar to Twitter and allows users to:

  • Create an account
  • Post, edit, and delete tweets
  • Follow other users
  • View tweets from the people they follow

The goal was to learn full-stack development and practice working with user data, routing, and basic state management.

Challenges and Areas for Improvement:

  • Performance: Tweets can take time to load. There is no lazy loading or pagination initially.
  • Mobile Experience: Not fully responsive on smaller screens; buttons and text may overlap.
  • User Control: No way to delete an account or report harmful content.

Ethical Considerations:

  • Data Privacy: Users should control their data and be able to delete their accounts.
  • Content Responsibility: A flag or report system is needed to manage harmful content.
  • Fair Usage: The app should include rate limits and explain what data is collected.

Refinements and Ethical Integration:
Since the initial build, I have refined the project by adding infinite scroll using the following React component integration: 

<InfiniteScroll
  dataLength={tweets.length}
  next={() => setPage((page) => page + 1)}
  hasMore={hasMore}
  loader={
    <div className="flex justify-center my-4 text-gray-500">
      <span className="animate-pulse">Loading more tweets...</span>
    </div>
  }
  scrollableTarget="tweet-main-content"
>

Additionally, I improved the responsiveness with CSS media queries to adapt the layout and hide non-essential elements on smaller screens, enhancing the mobile user experience.

Planned further improvements include adding user account deletion, a reporting system for harmful content, and rate limiting to ensure fair use. These steps emphasize my commitment to respecting users’ rights and fostering a safe online environment.

Spiritual Lessons Learned:
This project taught me how to balance technical deadlines with honesty and patience. By incorporating prayer and spiritual reflection throughout development, I strengthened my commitment to ethical practices and serving others through technology.

Conclusion:
This project helped me understand the technical side of web development deeply. It also opened my eyes to the importance of respecting users, protecting their data, and creating a safe and ethical experience. By making these changes, I aim to improve both the function and moral integrity of the app.

Tsogt’s SSP Project is a compact version of Twitter built using React.js and Node.js. Users can create accounts, post tweets, edit or delete them, follow others, and see posts from people they follow. It’s a solid foundation for a social media app, especially for something built solo a couple years back.

The goals are clearly stated, and the features are well thought out. It’s great that he’s already thinking about ways to improve things like performance and mobile responsiveness, it shows he’s not just stopping at “it works,” but wants to make it better.

2. Technical Assessment

From a technical perspective, it’s clear Tsogt knows what he’s doing. The use of React and Node is pretty standard for this kind of project, and it sounds like the core functionality is working well. The fact that he's already reflecting on how to make things faster and more mobile-friendly shows a good developer mindset.

Adding in some scalability improvements like lazy loading or pagination could help with performance. And throwing in responsive design tweaks would make it much smoother on phones.

3. Ethics and User Considerations

This part really stood out. A lot of people ignore ethics in small projects, but Tsogt’s already thinking about user data, content safety, and fair treatment. That’s a big deal.

He mentions things like letting users delete their data, avoiding unnecessary data collection, and letting people report harmful content. These are great considerations and totally in line with modern expectations, even if the app isn’t huge.

4. Suggestions and Feedback

  • Try adding pagination or infinite scroll to help load tweets faster.
  • Improve responsiveness so it works better on different screen sizes, especially phones.

Overall, this is a well-built, thoughtful project. Tsogt shows he’s got both the technical chops and the awareness to think about the real-world impact of what he builds. With a few updates, especially around mobile support and moderation tools, this could go from a good student project to something really polished.
