Project 3: SSP Project – Refinement and Ethical Integration

Author: Tsogt Enkhbat
Date: July 2025

GitHub Repository: https://github.com/heytoshi/SSP-Project

Project Overview:
The SSP Project is a simple social media app I built two years ago using React.js and Node.js. Inspired by Twitter, it allows users to:

  • Create an account
  • Post, edit, and delete tweets
  • Follow other users
  • View tweets from users they follow

The goal was to learn full-stack development and understand how user data and state management work in real applications.

Challenges and Areas for Improvement:

  • Performance: Tweets load slowly without lazy loading or pagination.
  • Mobile Experience: Not fully responsive on small screens.
  • User Control: No account deletion or harmful content reporting features.

Ethical Considerations:

  • Data Privacy: Users deserve control over their data, including deletion options.
  • Content Responsibility: Harmful content should be reportable by users.
  • Fair Usage: Rate limits should prevent abuse, and all data policies must be transparent.

Refinements and Ethical Integration:

Since the initial release, I’ve made key improvements. I added infinite scroll using React components:

<InfiniteScroll
  dataLength={tweets.length}
  next={() => setPage((page) => page + 1)}
  hasMore={hasMore}
  loader={
    <div className="flex justify-center my-4 text-gray-500">
      <span className="animate-pulse">Loading more tweets...</span>
    </div>
  }
  scrollableTarget="tweet-main-content"
/>

I also improved responsiveness using media queries to clean up the layout for mobile devices. No user tracking cookies are used. Planned updates include:

  • Adding a delete account option
  • Building a reporting system for unsafe content
  • Implementing rate limits for fair use

Each of these changes reflects a commitment to ethical design and user protection.

Spiritual Lessons Learned:

When deadlines got stressful, I was tempted to skip important ethical features. But through prayer, I was reminded that my responsibility is to build tools that serve and protect people. Like in Mosiah 2:17, I saw that when I serve users righteously, I’m also serving God.

Conclusion:

This project helped me grow technically and spiritually. I learned how to manage user data responsibly, prioritize ethical design, and let spiritual impressions guide my decision-making. These lessons continue to shape how I build software with integrity.
